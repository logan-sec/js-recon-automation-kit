# `har_recon.py`

`har_recon.py` is the main script in the JS Recon Automation Kit.

It takes a browser-exported `.har` file, extracts JavaScript responses, beautifies them, runs endpoint/secret discovery tools, and organizes the output into manual review buckets.

This script is built for **manual JavaScript recon**, not automatic bug finding.

---

## What It Does

Given a HAR file, the script will:

1. Extract JavaScript files from the HAR
2. Beautify the JavaScript
3. Run LinkFinder against each JS file
4. Run SecretFinder against each JS file
5. Run local regex-based triage checks
6. Save organized output into a recon folder

Example output:

```txt
recon_target/
├── js/
│   ├── 0001_app.js
│   └── 0002_dashboard.js
├── lf_out.txt
├── sf_out.txt
└── gf_out/
    ├── high_value_endpoints.txt
    ├── idor.txt
    ├── oauth_redirects.txt
    ├── authz.txt
    ├── secrets.txt
    ├── webhooks.txt
    └── admin_flags.txt
```

## Usage

Basic usage:

```bash
python3 scripts/har_recon.py target.har
```

Custom output folder:

```bash
python3 scripts/har_recon.py target.har --output recon_target
```

Skip LinkFinder:

```bash
python3 scripts/har_recon.py target.har --skip-lf
```

Skip SecretFinder:

```bash
python3 scripts/har_recon.py target.har --skip-sf
```

Skip local triage buckets:

```bash
python3 scripts/har_recon.py target.har --skip-gf
```

Custom LinkFinder and SecretFinder paths:

```bash
python3 scripts/har_recon.py target.har \
  --linkfinder ~/LinkFinder/linkfinder.py \
  --secretfinder ~/SecretFinder/SecretFinder.py
```

## How To Create a HAR File

1. Open the target in your browser
2. Open DevTools
3. Go to the Network tab
4. Enable Preserve log
5. Browse important app flows
6. Right-click the requests
7. Export as HAR
8. Run the HAR through this script

Useful flows to capture:

- Login
- Signup
- Account settings
- Billing
- Profile updates
- Upload/download features
- Integrations
- Webhooks
- OAuth/SSO flows
- Admin-like areas if available in scope

## Output Files

### `js/`

Contains beautified JavaScript files extracted from the HAR.

These are the files you manually review.

---

### `lf_out.txt`

Contains LinkFinder output.

This is useful for finding:

- API routes
- Hidden endpoints
- Internal-looking paths
- OAuth/callback URLs
- Upload/download routes
- GraphQL endpoints

Expect noise. LinkFinder output is a starting point, not a final result.

---

### `sf_out.txt`

Contains SecretFinder output.

This is useful for finding possible:

- API keys
- Tokens
- Cloud keys
- Auth-related strings
- Config values
- False positives

Do not assume anything in `sf_out.txt` is valid. Most output needs manual validation.

Only validate keys when the program rules allow it.

---

### `gf_out/`

Contains local triage buckets generated by regex matching.

These files are not confirmed vulnerabilities. They are leads.

Examples:

```txt
high_value_endpoints.txt
idor.txt
oauth_redirects.txt
authz.txt
secrets.txt
webhooks.txt
admin_flags.txt
```

Start with:

```txt
gf_out/high_value_endpoints.txt
gf_out/idor.txt
gf_out/authz.txt
gf_out/oauth_redirects.txt
gf_out/secrets.txt
lf_out.txt
sf_out.txt
```

---

## Important Note About Noise

Real output will usually be noisy.

You should expect:

- False positives
- Duplicate endpoints
- Framework strings
- Random library paths
- Low-value matches
- Minified JS artifacts
- Output that needs manual filtering

The value of this script is not perfect signal.

The value is that it organizes messy JavaScript recon into files that are easier to review.

## Troubleshooting

### No JS files found

Your HAR may not contain JavaScript responses.

Try again with:

- Preserve log enabled
- Cache disabled
- More app flows visited
- A logged-in session if allowed

---

### LinkFinder not found

Install LinkFinder or pass the path manually:

```bash
python3 scripts/har_recon.py target.har --linkfinder ~/LinkFinder/linkfinder.py
```

---

### SecretFinder not found

Install SecretFinder or pass the path manually:

```bash
python3 scripts/har_recon.py target.har --secretfinder ~/SecretFinder/SecretFinder.py
```

---

### `ModuleNotFoundError: jsbeautifier`

Run:

```bash
pip install -r requirements.txt
```

---

### Prettier not found

Prettier is optional.

Install it with:

```bash
sudo npm install -g prettier
```

If Prettier is missing, the script will still try to use `jsbeautifier`.

---

## Reminder

This script does not replace manual testing.

It helps you organize the mess so you can review JavaScript faster.
